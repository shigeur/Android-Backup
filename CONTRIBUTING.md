# Contributing to Android Backup

First off, thank you for considering contributing to Android Backup! It's people like you that make open-source projects such a great community.

## Where do I go from here?

If you've noticed a bug or have a feature request, please [open an issue](https://github.com/shigeur/Android-Backup/issues) to discuss it before submitting a pull request.

## Pull Requests

1. **Fork the Repository:** Create your own fork of the code.
2. **Create a Branch:** Create a branch for your feature or bug fix.
3. **Commit Your Changes:** Make your changes and commit them with descriptive commit messages.
4. **Test:** Please ensure the application still builds and runs correctly.
5. **Push and Open a PR:** Push your branch to your fork and submit a Pull Request to the `main` branch.

## Code Style

- We follow standard Swift and Apple HIG guidelines.
- Try to keep UI logic and business logic separate (MVVM architecture is preferred where possible).
- Use `SF Symbols` for icons wherever possible.

## Setup Development Environment

1. Clone your fork.
2. Ensure you have the latest version of Xcode installed.
3. Open `Package.swift` or the project folder in Xcode.
4. Build and Run!

If you need any help, feel free to ask in the issues section!
